# Epic 1 Context: Foundation & the Seeded Shelf

<!-- Generated from planning artifacts. Regenerate with compile-epic-context if planning docs change. -->

## Goal

Stand up the walking skeleton for PRESS START — a single-user, installable PWA that replaces a Notion game-tracking database — and deliver the read-only face of the product with real, trustworthy data. This epic scaffolds the Cloudflare project (Worker + React SPA + D1 + CI/CD), implements the pure domain core (effective/derived state, title normalization, invariant predicate), wires better-auth magic-link sign-in with day-one user scoping, defines the catalog/tracking data model behind a repository seam, builds the dark-only design-token system and responsive app shell, runs the one-time out-of-band seed import of the real library (~344 games), and renders a cover-forward, backlog-first shelf that can be scrolled and searched. Card flip-to-edit and status changes are deliberately deferred to Epic 2 — this shelf is read-only. Ties to the primary success metric: the library can be trusted, every game present without manual entry.

## Requirements & Constraints

- **State model.** Play status is one per game (`Not started`/`Up next`/`Playing`/`Paused`/`Dropped`), default `Not started`, nullable only once a completion milestone exists. Completion milestones (`completed_on`, `platinum_on`) are dates; non-null means achieved. Every game must always have a play status OR at least one milestone (invariant). `Dropped` is hidden from the default shelf.
- **Effective & derived state.** Effective state = play status if set, else Platinum if `platinum_on`, else Story completed if `completed_on`. Derived (never persisted): Released (real date ≤ today; TBA/missing = false), Wishlisted (= not owned), Playable now (= (owned OR in PS+ Extra) AND released). Ordering, card labels, and filters operate on effective state only, never raw play status.
- **Default shelf.** Shows only live-play-status games (Completed/Platinum/Dropped hidden — this is the backlog view). Order: Playing → Paused → Up next → Not started, alphabetical within each group. Infinite scroll with an always-visible name search bar that matches the entire library, ignoring active filters and hidden states.
- **Genre vocabulary** is sourced exclusively from IGDB; the seed re-tags all games. Adding a game with new genres auto-creates genre rows.
- **Seed import** loads the Notion CSV + PS library exports, enriches every game from IGDB (cover, genres, release date). Membership-sourced PS+ claim entries are excluded (never created, never `Owned`); skipped count reported. Title reconciliation is case-insensitive after stripping trademark glyphs, leading articles, edition suffixes, normalizing whitespace; PS4/PS5 duplicates collapse to one PS5 entry. Notion status mapping: `Completed`→null status + `completed_on` (from Date finished), `Up next!`→Up next, `Not released`→Not started, `Not started`/`Playing`/`Paused` 1:1, Date started→`started_on`, Rating not imported. `Owned: Yes` imports as owned (physical default). No fabricated dates — stamp only what the CSV knows; unplaceable/unmatched rows land in the stragglers list, never guessed.
- **Platform.** Installable PWA, one responsive app with desktop and phone both first-class. better-auth magic link (no passwords). All tracking data user-scoped from day one; no sharing/roles/tenancy.
- **Free-tier & no-external-on-render.** Free-tier hosting is a hard constraint; the app is stateless. Nothing external is fetched on a read/render path — covers and store links come only from persisted data; third-party APIs are hit only at import/sync/refresh/add time. Failures surface, never silently retry.

## Technical Decisions

- **Stack (pinned).** Cloudflare single-vendor: one Worker serves the React SPA (Workers Static Assets) AND the Hono JSON API; persistence is D1 via binding; scheduled work is Cron Triggers. Runtime is workerd/V8, TypeScript throughout; Bun is local-only (package manager, test runner, out-of-band scripts) — no Bun-only runtime APIs. Drizzle ORM 0.45.x + drizzle-kit, Hono (+ typed RPC client), Zod (shared SPA↔Worker at every boundary), TanStack Query, React + Vite + vite-plugin-pwa, better-auth (magic link), Vitest + `@cloudflare/vitest-pool-workers`, Biome v2.
- **Layering (ports-and-adapters, two seams).** `core/` is I/O-free (no fetch, no D1/Drizzle) — effective/derived state, normalization, reconciliation, invariant checks are pure and unit-tested without network/DB. All DB access goes through `repositories/` (Drizzle) — no raw D1 elsewhere; this is the swap point for a future DB migration. Every third-party call goes through a `providers/` adapter. Nothing-external-on-render is enforced structurally: a fetch in a query path is an architecture violation. Namespaces: `core/ services/ repositories/ providers/ routes/`.
- **Single-owner domain functions.** One `core/` function computes effective state (ordering/labels/filters consume it, none recompute). One `core/` title-normalizer produces the shared `title_normalized` match key (strip glyphs/edition suffixes, drop leading articles, case/whitespace-fold, PS4/PS5→PS5) — a non-unique candidate key. Notion status-mapping is a pure `core/` function.
- **Data model.** `GAME` holds shared catalog facts (title, `title_normalized`, release_date, cover_url, store_url, PS+ Extra membership per region, `unenriched`, genres via `GAME_GENRE`). `GAME_TRACKING` holds per-user state, PK `(user_id, game_id)`, GAME 1:many GAME_TRACKING; play_status nullable, milestone/lifecycle dates, owned, ownership_type. Identity is `EXTERNAL_LINK (source, external_id)`, many-per-(game, source) so PS4+PS5 ids → one GAME. `title_normalized` carries NO uniqueness constraint. Every query filters by `user_id`. Straggler = a needs-attention record (import staging rows carrying Notion payload, or name-only `unenriched` GAME rows). Epic-1 entities: `USER`, `GAME`, `GAME_TRACKING`, `GENRE`, `GAME_GENRE`, `EXTERNAL_LINK`, `IMPORT_STRAGGLER`. `SETTING` and other later-epic tables are NOT created yet (entity-as-needed).
- **Delivery & ops.** The one-time seed import (~344 games, exceeds the 50 external-subrequest budget) runs out-of-band as a script writing D1 via the D1 HTTP API / Wrangler using the shared Drizzle schema — no UI surface. Migrations run from CI, never at deploy/startup: `drizzle-kit generate` → `wrangler d1 migrations apply` before `wrangler deploy`. CI on every push/PR: Biome (lint+format) + `tsc` + Vitest (workers pool). One set of `package.json` scripts (`lint`/`typecheck`/`test`) is invoked identically by local dev, CI, and the bmad-loop `[verify]` gate so they cannot drift. Secrets via Wrangler; the live PSN cookie later lives in a D1 settings table (not this epic). D1 file, `.env`, `node_modules/` gitignored.
- **Shelf rendering.** At v1's single-user ~344-game scale the effective-state-sorted set is materialized in the Worker/client and rendered progressively (infinite scroll), NOT keyset-paged with a SQL `ORDER BY play_status`. Search uses a dedicated whole-library query, separate from the filtered-shelf query — not a client filter over the paginated shelf.

## UX & Interaction Patterns

- **Design tokens (dark-only, no light theme).** Colors: void `#05090f`, surface `#0b1622`, surface-raised `#0a1120`, borders hairline `#163043`/soft `#12283a`, text primary `#eafaff`/secondary `#8fb0c4`/muted `#6b8ba0` (muted floored here — do not go dimmer for text), brand-blue `#0070cc`, electric `#12b3ff` (everyday interactive neon), cyan glow `#35e0ff` (halo partner), heat magenta `#ff2e88` / ink `#ff8bc2` (reserved for Playing only, ~1–3 cards — never common flags), milestone silver `#d6e6f5`, state-dormant `#3d5566`, warn-amber `#ffb254`, success-green `#8fe6a8`. Spacing scale 4/8/12/16/24/32. Radii sm 8 / md 12 / lg 18 / pill 999.
- **Typography — four faces by job.** Orbitron (display: wordmark 900, headings 700, pills/labels 600 uppercase tracked, card titles single-line ellipsis — never wrap); Rajdhani (condensed UI labels: buttons, segmented controls, dropdowns); Inter (body/detail/forms); JetBrains Mono (numerals, dates, counts, timestamps, tagline, filter-logic labels). Card title 12.5/600 ellipsis; pill label 10/600 uppercase ~.04em; body 13/Inter. Semantic state colors: Playing = magenta, Up next = cyan/electric, Paused = steel `#8fb0c4`, Not started = dormant grey `#3d5566`, Completed/Platinum = silver.
- **Brand.** Wordmark "PRESS START" (Orbitron 900, neon glow, blinking cursor) + tagline "Want it! Own it! Beat it!" (JetBrains Mono, tracked). Void background carries a faint Tron light-grid + subtle blue→magenta radial wash behind the shelf. Hard legal rule: never use PlayStation/Sony marks in branding/chrome — only as descriptive text. Personality is dialled light — arcade flavour lives in empty-state headers and the wordmark; copy stays plain elsewhere. No gamification (XP/streaks/badges), no ratings.
- **Elevation via glow and tone, not drop-shadow.** Cards on `surface`; modals/popovers on `surface-raised` with a cyan glow-ring (`0 0 22–34px` accent-glow, low alpha); the Playing card carries a soft magenta bloom; focus/selection as neon halos.
- **App-shell primitives (built here, fed by later stories).** Reusable Attention-banner (full-width under-header; amber = stragglers, magenta = expired cookie, steel = failed refresh; persistent until the condition self-resolves), Toast (transient bottom, surface-raised cyan-edged, ~3s, UNDO variant), Skeleton (cover-shaped shimmer), and a polite live region for announcements. Empty states: no filter match → `NO MATCH` + "Clear filters"; empty library → `INSERT GAMES` + "Sync library"/"＋ Add a game".
- **Card (read-only in this epic).** Cover-forward (3:4); top-left display-only flag cluster (PS+ Extra badge ◈ when in-catalog & not owned, release-state flag TBA/upcoming until released, milestone badge silver ✓/🏆 persisting regardless of play status); info strip below (name Orbitron ellipsis, effective-state status pill, genres desktop-only); Playing card glows magenta.
- **Responsive deltas.** Phone: 2-up lean card, genres hidden, single Filters button, icons-only FAB, full-screen detail, compact header ("175 · 52 OWNED"); bottom-pinned search. Desktop: auto-fill dense grid (~150px min), full filter row + inline summary, icons+text FAB, ~760px detail panel, full header with `PS+ CATALOG AS OF …`; header-left search. PWA installable with home-screen icon; no offline requirement.
- **Accessibility floor.** Focusable shelf grid with arrow traversal in reading order. Distinct always-on focus outline (never glow-intensity alone). WCAG AA contrast (≥4.5:1 body, ≥3:1 large/UI); status pills use translucent tint + light ink or dark ink on neon — never white-on-neon. No color-alone signaling — the text status pill is the indicator on every surface (dot-only treatment is superseded, must not ship). Accessible names + state for icon-only controls; ≥44×44 touch/hit targets decoupled from visual size (padding/invisible expander). `prefers-reduced-motion`: replace flip-then-grow with fast cross-fade/scale, drop glow pulses and skeleton shimmer (static placeholder); static neon stays. Search bar has combobox semantics and a global focus shortcut.

## Cross-Story Dependencies

- 1.1 (scaffold, CI/CD, layer namespaces) underpins every other story.
- 1.2 (`core/` effective/derived state + normalizer) is consumed by 1.6 (reconciliation, status mapping) and 1.7 (shelf ordering/labels). The invariant predicate is defined here but its enforcement is wired in Epic 2.
- 1.3 (auth + `user_id`) and 1.4 (schema + repositories) both underpin 1.6 (writes through the schema) and 1.7 (user-scoped reads through repositories).
- 1.5 (tokens + shell primitives) is the visual foundation 1.7's shelf and all later surfaces render in.
- 1.6 must run before 1.7 shows real data.
- Deferred to Epic 2: card flip-to-detail, status changes, milestone logging, invariant enforcement, lifecycle-date stamping on transitions. Deferred later: `SETTING` table + PSN cookie (Epic 4), PS+ Extra membership realizing Playable-now (Epic 5), add-by-name + straggler resolution UI + CSV export (Epic 6).
