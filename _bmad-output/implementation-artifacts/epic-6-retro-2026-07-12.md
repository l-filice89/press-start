## Significant discovery — cross-tree signalling is now load-bearing IA

The window-event pattern (`OPEN_DETAIL` / `SEED_SEARCH` / `SHELF_SEARCH`) started as the smallest channel between sibling components and is now the app's de-facto cross-tree bus. It has repeatedly produced mount-race and duplicate-surface bugs, and forced a whole search redesign to resolve a contradiction it enabled. **Epic 7 adds a NEW navigation destination (catalog browse) — a second place besides the shelf.** If 7.x accretes more `CustomEvent`s it inherits the failure modes. Story 7.0 (already an arch + UX design gate) is the right place to decide the cross-tree communication approach deliberately — router/context vs more window events — rather than adding another event under load. **Resolution:** folded into 7.0's scope as an explicit design input (action item 2).

1. **The follow-up-review gate is proven.** Epic 5 called it unproven; Epic 6 fired it on 6.4/6.5, an independent pass found and fixed real write/undo/a11y issues, and the flags were consumed. Next step hardens it: auto-force on HIGH (item 1).

4. **Deferrals are clustering — give the clusters owners.** `candidates[0]` (item 3) and multi-user blockers (item 4) each appeared 2–3× as "rare, single-user." Correct per story; the retro is where the cluster gets a home instead of a fourth deferral.
5. **Fifth consecutive full follow-through.** Every Epic-5 item closed, including the degenerate-response rule that then protected Epic 6's IGDB work in the same epic it was written for.
