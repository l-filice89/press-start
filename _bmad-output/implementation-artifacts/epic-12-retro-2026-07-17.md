## What went well

- **UI-MOCK-GATE proved itself on its first outing (Luca's headline).** The desktop-crowding risk was named in planning, routed to the mock stage, resolved with a sign-off — and **zero UI adjustments were needed after the code was written**. The rule pipeline compounding: Epic 10's retro wrote the rule, Epic 12's planning aimed a known risk at it, the gate closed it before code.
- **Speed.** Correct-course to shipped-and-independently-reviewed in one day; Luca: "Love the speed that BMAD allows to achieve."
- **Single-predicate discipline held.** The Time group is one clause in `applyShelfFilter` — no parallel filtering path; Epic 3's filter system absorbed a new dimension without reopening.
- **The absence contract got stronger under review.** Spec said `null` ⇒ Unknown; review hardened the guard to route `undefined`/`NaN`/negatives to Unknown too, with hazard tests — 10.3's contract extended honestly into filtering.
- **Evidence-honesty continued** — COVERAGE.md rows corrected to match what the tests actually prove.

## Challenges / lessons

1. **A11y surfaced in review, second epic running.** The metric toggle shipped mouse-only and ARIA-invalid (plain buttons inside `role="menu"`, outside the roving focus); the fix reworked shared `FilterDropdown` machinery mid-review, forcing the follow-up pass (which came back clean). 10.5 had a11y-shaped patches too. Luca's ruling: **the net is working — no new gate**, but a spec-level nudge is cheap: interactive widgets name their ARIA pattern up front, and the app must remain WCAG compliant. → Action item 1 (shipped this session).
2. **Deferred copy decision resolved at the retro:** the summary sentence didn't name the active metric ("≤25h" read identically under story or 100% hours). Luca picked the copy — "… story completion" / "… 100% completion" — shipped via quick-dev one-shot this session. → Action item 2.
3. **No friction reported.** One-day epic, nothing felt slow.

## Key takeaways

1. **A gate that fires before code beats a review that fires after.** UI-MOCK-GATE's first live test eliminated the build→react→rework loop entirely — zero post-implementation UI changes.
2. **Small epics ride the accumulated rails.** One story shipped same-day at low severity because eleven epics of standing rules, e2e conventions, and review contracts were already in place.
3. **Shared-component rework is the real risk in small UI stories** — the one medium finding forced a rework of machinery under State/Genre; the FOLLOW-UP-REVIEW CONTRACT existed for exactly this and cleared it.
4. **Retros are a fine place to ship small decisions, not just record them** — the copy fix and the WCAG nudge both landed inside this session.
