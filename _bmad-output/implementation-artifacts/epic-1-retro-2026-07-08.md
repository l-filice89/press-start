# Epic 1 Retrospective: Foundation & the Seeded Shelf

Date: 2026-07-08
Participants: Amelia (Developer, facilitating), John (Product Manager), Winston (Architect), Sally (UX Designer), Murat (Test Architect), Luca (Project Lead)

## Epic Summary

- **Completed:** 7/7 stories (100%)
- **Duration:** ~4 days wall-clock (2026-07-04 → 2026-07-08)
- **Interruptions:** 2 dev-session timeout/rollback cycles on the heaviest UI stories (1.5, 1.7) — resolved going forward by a subscription upgrade giving more context per session; 1 external rate-limit stall (1.6)
- **Production incidents:** 3 total
  - `490de6b` — API routes shadowed by the SPA fallback (a hazard Story 1.1's own AC explicitly named) + a genre-lookup perf/quota fix
  - `558afe7` — seed import wrongly excluded some PS+ claims; IGDB title-matching gaps
  - **Production auth outage** (2026-07-08, mid-retro) — `BETTER_AUTH_SECRET`/`RESEND_API_KEY` were never synced from GitHub Actions secrets into the Cloudflare Worker, so every `/api/auth/*` route 500'd. Caught live, root-caused, and fixed within the same session (commits `94dc679`, `27f430f`).
- **Business outcome:** Success metric #1 ("the library can be trusted, every game present without manual entry") is delivered — the seed import landed the full 344-game library day one — but two of the three incidents above directly touched this metric before being caught.

## What Went Well

- Seed import delivered the entire real library on day one, proving success metric #1 before the shelf UI even existed.
- Story 1.4 (data model + repositories) ran clean in a single pass — well-scoped, low-ambiguity, review's own risk heuristic judged it low-risk and skipped a second look. The shape other stories should aim for.
- Story 1.7 folded its review-hardening into the same commit, pre-merge (`25a383a`), rather than letting issues leak into a post-merge hotfix — the containment pattern to repeat.
- Story 1.3 (auth) was done manually rather than via the autonomous bmad-loop — the right call for a security-sensitive story, not a gap.
- The end result matched expectations exactly: a read-only shelf behind auth, no scope renegotiation mid-epic. The Epic 1 deferrals written up front (flip-to-detail, status changes → Epic 2) held without drift.

## Challenges

- Two heavy-UI stories (1.5, 1.7) blew the dev-session timeout on first attempt, needing a `git reset --hard` rollback and a second run. **Resolved** — subscription upgrade increased context per session.
- A routing hazard named explicitly in Story 1.1's own acceptance criteria, and a concern Luca raised live during Epic 1 development, were both downplayed rather than gated — both later surfaced as post-"done" hotfixes.
- Story 1.5 was flagged `followup_review_recommended: true` by the loop's own review gate and shipped without the follow-up happening — a review signal silently dropped, same shape as the routing-hazard miss above.
- **Production auth outage:** GitHub Actions secrets and Cloudflare Worker secrets are separate stores; `wrangler deploy` never bridges them. `BETTER_AUTH_SECRET` was set locally (`.dev.vars`) but never pushed to the Worker in production, so every auth route failed uniformly. Same root pattern as the other two incidents: a real hazard, no automated check, shipped anyway, only caught by manual discovery (Luca hitting the login page).
- Current testing is pass/fail on the suite as a whole, not proof against specific named hazards — the gap that let all three incidents through.

## Key Insight

Three separate incidents this epic — API routing, seed-import exclusion logic, and the secrets-sync gap — share one root cause: **a hazard was known (named in an AC, flagged by review, or raised live) and nothing automated gated on it.** Discovery happened by production use, not by test.

## Action Items

**Process:**

1. **Wire a "hazard-named-AC gets a hazard-named-test" rule into the bmad-loop dev/review gate** — any acceptance criterion naming a specific invariant or failure mode requires a test asserting exactly that behavior before a story reaches `done`.
   Owner: Murat (rule definition) + Winston (wiring into `.bmad-loop/policy.toml`)
   Status: done — added as a `persistent_fact` in `_bmad/custom/bmad-dev-auto.toml`, loaded into every dev-auto session (not a `policy.toml` key: the rule needs semantic judgment, which `[verify]`'s deterministic commands can't express)

2. **Close the loop on `followup_review_recommended: true`** — a story cannot reach `done` with that flag set until the follow-up review happens or is explicitly and visibly declined in the record.
   Owner: Amelia (loop gate logic)
   Status: done — `.bmad-loop/policy.toml` `[review] trigger` changed from `"recommended"` to `"always"`: every story now gets an independent second-opinion review pass, removing dependence on the dev pass's own self-assessment (which is exactly where Story 1.5 fell through — it *did* flag `followup_review_recommended: true` and the follow-up session still missed the routing hazard)

3. **Sync GitHub Actions secrets to the Cloudflare Worker on every deploy, and smoke-test the real auth endpoint.** Already implemented and verified live during this retro.
   Owner: Amelia / Luca
   Status: done (commits `94dc679`, `27f430f`)

**Team Agreement:**

- Manual-human-loop carve-outs (like Story 1.3's auth) are named exceptions against the full-autonomy goal, not a silent default.
- New standing target: **bmad-loop should complete a full epic with zero human interaction and still deliver the intended result.** Today's manual-auth exception and the three incidents above are the debt against that goal, not permanent fixtures.
