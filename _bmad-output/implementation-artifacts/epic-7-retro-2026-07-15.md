- **The immune system, not the infection count.** 60 defects / 16 high all died before main; highs reaching production remain **zero**. Luca's framing adopted: the number to manage is escapes, not catches.
- **The 7.0 design gate earned its cost twice.** The live store probe (no release_date, facet-only genres) prevented modeling fiction; the router decision made 7.2's migration a plan instead of a firefight.

2. **Green-suite false confidence, three flavors in one epic:** a shipped test that *encoded* a data-loss bug (7.1 H1); e2e pinning the degraded no-IGDB-creds path as the happy path (7.3 H3); and `web/**/*.test.ts` matching **no** vitest project since Epic 3 (`filters.test.ts` never ran — found 2026-07-15). → Action item 2 (guard shipped).
3. **The lock/budget/generation protocol is the epic's complexity center.** Most highs cluster on the single-flight lock, the 50-subrequest ledger, and generation fencing. The Epic 9 rules (BUDGET-COUNTS-EVERY-SUBREQUEST, CAPABILITY-IS-NOT-AN-IDENTIFIER, TEST-THE-BYPASS) all fired here — they are load-bearing, not ceremony.

## Key takeaways

1. **A delegation without an AC is a gap with no owner.** Reviews audit diffs; nobody audits the space between stories. The new rule makes the receiving spec the owner-of-record.
2. **A green suite can lie three different ways** — a test encoding the bug, a test pinning the degraded path, and a test that never runs. Each needs a different defense; two shipped as rules/guards today.
3. **Catch-count is not a quality problem when escape-count is zero.** 16 pre-ship highs on an epic touching a live store, a shared lock, and a router migration is the review architecture working as designed.
4. **Design gates keep paying.** 7.0 (probe + router decision) is two epics of evidence that deciding structure before code beats deciding it under load.
5. **Sixth consecutive full follow-through** on retro commitments — including the two Epic 6 items that shaped this epic's best outcomes.
