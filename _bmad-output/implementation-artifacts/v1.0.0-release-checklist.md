# v1.0.0 Release Checklist — PRESS START (`ps-game-catalog`)

Go-live gate for the first tagged release. App is feature-complete (Epics 1–6
`done`); this checklist covers the merge, deploy, and release hygiene.

> **Publishing a GitHub Release triggers the production deploy.** As of
> 2026-07-13 `.github/workflows/deploy.yml` fires on `release: published` (plus
> manual `workflow_dispatch`) — **not** on push to `main`, so a merged PR alone
> ships nothing, and pushing the tag alone ships nothing either. The release's
> tag is the deployed ref. The job runs: lint → typecheck → test → build → D1
> migrate → deploy → smoke-test `/api/health` + `/api/auth/get-session`.
>
> Historical note: v1.0.0 and v1.1.0 shipped under the older push-to-`main`
> trigger, so the deploy steps ticked below happened at merge time, not at
> release-publish time.
