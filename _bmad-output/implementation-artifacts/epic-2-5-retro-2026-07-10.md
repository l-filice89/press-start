## Challenges / Patterns

1. **Vacuous e2e tests were the dominant review catch-class** — assertions that pass without proving (loading races, tautological residue checks, absence checks on unmounted panels). A failure mode specific to e2e; reviews caught it every story.
2. **Windows + SQLite friction became infrastructure:** 8K cmdline limit → `--file` execution; wrangler-CLI vs running-Worker SQLITE_BUSY → gated `POST /api/e2e/sql` Worker hook (2.5.3).
3. **`playwright/` was unlinted until 2.5.3** — a whole tier outside the Biome gate; now included with `noFloatingPromises: error`. Lesson: new source roots join the lint gate on creation.
4. **Process rules need adversarial review like code** — the standing rule's first draft had self-judged definitions and skip loopholes.

## Decisions

- **`/api/e2e/sql` posture accepted as-is:** arbitrary-SQL test hook gated by two independent factors (`E2E_TEST_HOOKS` defined only in the local e2e env + loopback-hostname check; 404 body mirrors the catch-all). Revisit trigger: repo goes public or a second deploy environment appears.
- **Standing-rule enforcement stays prompt-level:** trace compliance through Epic 3; build a mechanical COVERAGE.md audit only if a leak appears (action item).
- **Branch protection verified live** (`gh api`): required check `CI OK`, `strict: true`, `enforce_admins: true`, force-push/deletion blocked. No action needed.
