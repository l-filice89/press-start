## Significant Discovery — reveal-pill semantics change

Additive reveal semantics (hidden games OR into default set) push revealed games behind the FR-18 order + infinite scroll — reachable but a hassle (Luca). **Decision: reveal pills become an exclusive group.**

New semantics (locked 2026-07-10):

- Dotted pill active → **replaces the State group entirely** (state pills clear); visible set = only games matching selected hidden states.
- Dotted pills **OR among themselves** (Completed + Platinum = either).
- **Genre and Flag groups still AND** with the exclusive view.

Impact: FR-4/FR-20 + epics.md amendment required; `summarizeFilter` simplifies ("Showing Dropped games." becomes literally true — the 3.3 truthfulness patch becomes unnecessary); `epic3-reveal` + summary e2e rewritten to new contract. Implement as **Story 3.5 before Epic 4 kickoff**.

## Key takeaways

1. The deferred-work ledger works: 3.4 swept its assigned bundle cleanly; the six open items are visibility, not sloppiness — and two graduated to action items exactly when their risk profile changed (Epic 4 automated writes).
2. Zero-high epic: review severity is trending down while patch volume tracks surface area — the standing rules (failures-surface, hazard-named tests, Playwright coverage) are absorbing whole defect classes.
3. Copied chrome drifts — share one implementation (trap dedup, `loadAllPages`, `IN_FLIGHT`) instead of replicating.
4. Semantics that fight the ordering invariant (additive reveals vs FR-18 + infinite scroll) surface as UX friction, not bugs — catch them by using the feature, which is exactly how Luca caught this one.
